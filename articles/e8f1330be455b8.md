## この記事のゴール
この記事では、HubSpot × Teams 連携のうち、**「Microsoft Teamsのミーティングのレコーディングと文字起こしをHubSpotに同期する」機能**について設定方法や同期方法についてまとめたいと思います 🖇️✩ ₊˚

https://knowledge.hubspot.com/ja/integrations/sync-microsoft-teams-meeting-recordings-and-transcripts-to-hubspot

この連携機能を用いることで、Teams 会議のレコーディングやトランスクリプト（文字起こし）を HubSpot コンタクトのタイムラインに表示させるようにすることができます



Teams 側だと、レコーディングにアクセスできるのは基本的に会議に参加したユーザーのみとなるので（アクセス権を変更しない限り）、レコーディングのデータやそこから得られた情報は会議を行った担当者に留まってしまう状況になりがち… ↴↴

この連携機能を使うと👇のように、
Teams 会議のレコーディング・トランスクリプトが HubSpot 側に同期され、**「コンタクトと過去どのような会議を行っていたか？」** を HubSpot から振り返ることができるようになります 💫⟡.·

![](/images/e8f1330be455b8/2025-12-31-17-33-18.png)

## 前提条件
本連携機能を用いるには、以下のセットアップが必要です：

1. Teams 連携をフルインストール方法で、HubSpot に接続している

連携機能のインストール方法については👇の記事をご参照ください 🐇୭
https://zenn.dev/raynya/articles/hubspot-teams-setup

2. アプリケーション アクセス ポリシーで、1.の連携アプリが Teams 会議のリソースにアクセスできるようにする

Teams のアプリケーション アクセス ポリシーで、HubSpot が Teams 会議のリソース（レコーディング・トランスクリプト）にアクセスができるように設定する必要があります。

📄∘˙ 詳細については、以下セクションにて・・・

また、ナレッジベースでは👇の記載もありますが
Teams クライアント や Outlook から Teams 会議を作成する際は予定表と会議は紐づいた状態で作成されますので、**特に気にしなくてOK**です 🫧=+.ﾟ

> 注: ミーティングのレコーディングと文字起こしをHubSpotに同期するには、ミーティングに対応するMicrosoft TeamsまたはMicrosoft Outlookのカレンダーイベントが必要です。カレンダーイベントがない場合、HubSpotはレコーディングデータと文字起こしのデータを取得できません。 [Microsoft Teamsのミーティングのレコーディングと文字起こしをHubSpotに同期する](https://knowledge.hubspot.com/ja/integrations/sync-microsoft-teams-meeting-recordings-and-transcripts-to-hubspot#%E5%A7%8B%E3%82%81%E3%82%8B%E5%89%8D%E3%81%AB)

:::details 細かいことをいうと… （会議を Graph API で作成する場合）
Teams 会議を Graph API で作る際、`POST /users/{userId}/onlineMeetings` のリクエストを使うことができます。

ただこの場合、公開情報に記載があるように
作成された会議はユーザーの予定表には紐づか**ない**形となります。
>この API は、ユーザーの予定表のイベントに関連付けられていないスタンドアロン会議を作成します。そのため、この API を介して作成された会議はユーザーの予定表には表示されません。[onlineMeeting を作成する](https://learn.microsoft.com/ja-jp/graph/api/application-post-onlinemeetings?view=graph-rest-1.0&tabs=http)


そのため、Graph API での作成を前提としている場合は イベントを作成する `/events` のリクエストを用い、
[リクエスト内で `"isOnlineMeeting": true` を指定しオンライン会議となるように作成ください](https://learn.microsoft.com/ja-jp/graph/api/user-post-events?view=graph-rest-1.0&tabs=http#example-4-create-and-enable-an-event-as-an-online-meeting) 🐈‍⬛⸝⋆
:::

### アプリケーション アクセス ポリシーについて
Teams のアプリケーション アクセス ポリシーは、ポリシーで指定されたアプリがユーザーに代わって Teams 会議にアクセスができるようにする設定となります。

https://learn.microsoft.com/ja-jp/graph/cloud-communication-online-meeting-application-access-policy

:::message
アプリはユーザーに代わって、Teams 会議にアクセスするため、ポリシーに割り当てられるユーザーがアクセスできる範囲の会議を読み取る状態となります 🐰 ∘°

>アプリケーションは任意のユーザーの代わりに動作できます。 そのため、ユーザーがオンライン会議にアクセスするために必要な権限 (開催者や共同開催者など) を持っていることを確認することが重要です。[アプリケーションがユーザーの代わりにオンライン会議にアクセスできるようにする](https://learn.microsoft.com/ja-jp/graph/cloud-communication-online-meeting-application-access-policy#allow-applications-to-access-online-meetings-on-behalf-of-a-user)
:::


今回は Entra 管理センター > エンタープライズ アプリケーション にある 連携用にインストールされている `HubSpot` のアプリ (アプリケーション ID :  "0493df14-0cd5-4c89-b413-e0b9b90bbb17") が、Teams 会議にアクセスできるように設定します:

![](/images/e8f1330be455b8/2025-12-31-15-29-20.png)

:::message
この設定は現在 **PowerShell でのみ** 設定が可能な項目となります。

またこの設定については、**管理者権限が必要**となりますので、Teams / テナント管理者に設定を依頼するようにしましょう 🐰 ∘°
:::


**<設定手順>**
基本的な手順は [ナレッジベース](https://knowledge.hubspot.com/ja/integrations/sync-microsoft-teams-meeting-recordings-and-transcripts-to-hubspot#%E5%A7%8B%E3%82%81%E3%82%8B%E5%89%8D%E3%81%AB:~:text=PowerShell%20%E3%82%92%E3%82%BB%E3%83%83%E3%83%88%E3%82%A2%E3%83%83%E3%83%97%E3%81%97%E3%81%A6%20Microsoft%20Teams%20%E3%81%AB%E6%8E%A5%E7%B6%9A%E3%81%99%E3%82%8B) にもありますが、ちょこちょこ補足します 🐝🌸◦°

1. PowerShell で `Connect-MicrosoftTeams` を実行、管理者権限を持つユーザーでサインインします
2. アプリケーション アクセス ポリシーを作成する
`New-CsApplicationAccessPolicy` コマンドを用い、アプリケーション アクセス ポリシーを作成します。

このとき、
- Identity : ポリシー名
- AppIds : ポリシー対象となるアプリ ID 。`HubSpot` のアプリ (アプリケーション ID :  "0493df14-0cd5-4c89-b413-e0b9b90bbb17") を指定します
- Description : ポリシーの説明
を指定します

// 例
```powershell
New-CsApplicationAccessPolicy -Identity ApplicationAccessPolicyforHubSpot -AppIds "0493df14-0cd5-4c89-b413-e0b9b90bbb17" -Description "This app aplication policy is created for HubSpot integration"
```
![](/images/e8f1330be455b8/2025-12-31-15-41-30.png)

3. 作成したポリシーをユーザーに割り当てる
`Grant-CsApplicationAccessPolicy` コマンドを用い、2 で作成したポリシーをユーザーに割り当てます。

このとき、
- 特定のユーザーに割り当てる
- テナント内の既存ユーザーすべてに割り当てる
- まだアプリケーションアクセスポリシーが割り当てられていないユーザーに割り当てる（Global の状態）
で指定方法が少し異なります。

https://learn.microsoft.com/en-us/powershell/module/microsoftteams/grant-csapplicationaccesspolicy?view=teams-ps

例えば、特定のユーザーに対して 2 で作成したポリシーを割り当てる場合は以下のように設定します 🪄︎︎◝✩

// 例
```powershell
Grant-CsApplicationAccessPolicy -Identity "User の ObjectID" -PolicyName "ApplicationAccessPolicyforHubSpot"
```


4. 設定結果を確認する
手順 2，3 で設定した内容を確認しましょう ५✍️

// 手順 2 で作成したポリシーを確認する
```powershell
Get-CsApplicationAccessPolicy -Identity "ApplicationAccessPolicyforHubSpot"
```

// 手順 3 でユーザーに割り当てたポリシーを確認する
```powershell
Get-CsUserPolicyAssignment -Identity "User の ObjectID" 
```
指定したユーザーに "ApplicationAccessPolicyforHubSpot" のポリシーが無事割り当てられていることが確認できました .ᐟ.ᐟ⭐️
![](/images/e8f1330be455b8/2025-12-31-15-52-15.png)



## Teams 会議のレコーディング・トランスクリプトを HubSpot に同期しよう
これにて、同期を行うための前準備は完了です ‧₊ 🍥 ˚✩ 
先述の ①、② の設定が完了すると、 HubSpot > [接続されたアプリ] > [Microsoft Teams] の [ミーティング] タブで、同期設定を行うことができるようになります。

### HubSpot 側の同期設定
[ミーティング] タブから、ポータル単位・ユーザー単位での同期設定を行うことが可能です。

[グローバル設定] では、**社内ユーザーがコンタクトとして取り込まれないようドメインの除外設定を行うことができます**ので、ここは事前に設定しておきましょう ५✍️

📄∘˙ 設定方法は以下ナレッジベースにあります：
https://knowledge.hubspot.com/ja/integrations/sync-microsoft-teams-meeting-recordings-and-transcripts-to-hubspot#microsoft-teams%E3%81%AE%E3%83%9F%E3%83%BC%E3%83%86%E3%82%A3%E3%83%B3%E3%82%B0%E5%90%8C%E6%9C%9F%E8%A8%AD%E5%AE%9A%E3%82%92%E7%AE%A1%E7%90%86%E3%81%99%E3%82%8B

### 自動同期
上記の設定画面 [自分の設定] > [ミーティングのレコーディングおよび文字起こしを同期] が有効になっている場合、その HubSpot ユーザーが（その HubSpot ユーザーが紐づけられている Teams ユーザー）行った Teams 会議のレコーディング・トランスクリプトは自動的に HubSpot に同期されるようになります。

同期されたレコーディングは [コール] ページに表示され、コールは **E メールアドレスに基づいてコンタクトレコードに紐づけ（もしくは新規作成）** がされます 🐬ྀིྀི‹₊˚

![](/images/e8f1330be455b8/2025-12-31-17-17-29.png)


また、Sales / Service Hub のシートが割り当てられているユーザーでは [コミュニケーションインテリジェンス(CI)](https://knowledge.hubspot.com/ja/calling/manage-phone-numbers-registered-for-calling?hubs_content=knowledge.hubspot.com/calling/review-call-recordings-and-transcripts&hubs_content-cta=Conversation+Intelligence+%28CI%29#%E3%82%B3%E3%83%9F%E3%83%A5%E3%83%8B%E3%82%B1%E3%83%BC%E3%82%B7%E3%83%A7%E3%83%B3%E3%82%A4%E3%83%B3%E3%83%86%E3%83%AA%E3%82%B8%E3%82%A7%E3%83%B3%E3%82%B9%E3%82%92%E6%9C%89%E5%8A%B9%E3%81%AB%E3%81%99%E3%82%8B-%E3%80%8Csales-hub-%E3%80%8D%E3%81%BE%E3%81%9F%E3%81%AF%E3%80%8Cservice-hub-professional-%E3%80%8D-%E3%80%8Centerprise-%E3%80%8D%E3%81%AE%E3%81%BF) を用いた AI 分析を確認することも可能です。
![](/images/e8f1330be455b8/2025-12-31-17-12-00.png)


:::message alert
**⩩ 🐇 Ꮺ ⋆ 自動同期について**
検証している中で、いくつか注意点がありました…

以下の場合、会議は自動同期されません：
- Teams 会議の参加者のメールアドレスが、HubSpot ユーザーで使用されているメールアドレスと同じ場合
- Teams 会議の参加者が匿名ユーザーとして参加した場合（メールアドレスを識別しない状態になるため、コンタクトと紐づけがされないのかなと）

*ただ、匿名ユーザーの他に Teams にサインインした参加者がいた場合は、匿名ユーザーはコンタクトとして作成されず、メールアドレスが識別された参加者がコンタクトとして紐づいていました

また、ナレッジベースには「[Microsoft Teamsミーティングが終了すると、HubSpotアカウントのコール インデックス ページにコールレコードが作成されます](https://knowledge.hubspot.com/ja/integrations/sync-microsoft-teams-meeting-recordings-and-transcripts-to-hubspot#microsoft-teams%E3%81%AE%E3%83%9F%E3%83%BC%E3%83%86%E3%82%A3%E3%83%B3%E3%82%B0%E3%81%AE%E3%83%AC%E3%82%B3%E3%83%BC%E3%83%87%E3%82%A3%E3%83%B3%E3%82%B0%E3%81%A8%E6%96%87%E5%AD%97%E8%B5%B7%E3%81%93%E3%81%97%E3%82%92%E8%A1%A8%E7%A4%BA%E3%81%99%E3%82%8B)」との記載がありましたが、
Teams 会議の招待時に指定した終了時間**以降**でないと、コンタクトの作成やレコーディングの同期は行われないようでした。

*例えば、👇のコンタクトは 16 時台に会議が行われていましたが、会議は 17:00 に終わるようにスケジュールされており、コンタクトの作成（作成によるコールの同期も）は 17:00 に行われたとされています：
![](/images/e8f1330be455b8/2025-12-31-17-25-31.png)
:::

ちなみに、コール同期で作成されたコンタクトのソースは以下の通りになります：

| レコードソース | レコードソースの詳細 | オリジナル トラフィックソース | ドリルダウン 1 | ドリルダウン 2 |
| ---- | ---- | ---- | ---- | ---- |
| 連携 | Microsoft Teams | オフラインソース | INTEGRATION | Microsoft Teams |

### 手動同期
レコーディングが自動同期されなかった場合、手動で同期させることも可能です 𓏼˙₊🕊️◝ྀ

手動で同期をさせるには、[接続されたアプリ] > [Microsoft Teams] > [ミーティング] > [自分の設定] > [個別のミーティングを管理] で **ミーティング参加時に使う ID** を指定します：

![](/images/e8f1330be455b8/2025-12-31-16-56-33.png)

対象のミーティングが見つかったら、[ミーティングを同期] をクリックし同期が成功するのを待ちます ⋆ 🪼ྀི ⋆

同期が成功すると👇のように、[コール] ページに表示されるようになります
![](/images/e8f1330be455b8/2025-12-31-16-57-43.png)
