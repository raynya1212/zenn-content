## やってみよー
今回の手順としては以下の5ステップになります：
1. GraphQLのクエリを作成する
2. テーマを作成する
3. クエリをテーマ内に設置
4. テンプレートを作成する
5. テンプレートでクエリを呼び出す

:::message
今回はチュートリアル通りにテーマとテンプレートも新たに作っていますが、使用するテーマが既に構築されている場合はスキップしてください
:::

## 1.GraphQLのクエリを作成する
ページに呼び出したいデータのクエリを用意します。
ここは使用されているプランによって入り口が変わります：
- Sales Hub or Service Hub Enterprise がある場合
  - HubSpotのナビゲーションに［CRM開発］ってのがいるはずなので、そこから［GraphiQL］に進みます
- Content Hub Professinal以上の場合
  - ［CRM開発］機能がないので、[このドキュメント](https://developers.hubspot.com/docs/cms/data/query-hubspot-data-using-graphql#test-and-run-queries-interactively-using-graphiql)から GraphiQL tool にアクセスします

![](/images/bb90d880c8890c/2024-08-15-15-28-10.png)
*分かりづらいけどココ*

いずれかの方法にてGraphiQL toolにたどり着けたら、クエリを用意したいと思います。
方法としてはページに呼び出したい項目をポチポチしていけば自動でクエリを作ってくれるので、
"p_bands(カスタムオブジェクト名)_collection"というのを見つけて、itemsの中にある
- _metadata > id
- band_name *
- best_sold_album
- description
にチェックをいれます。

**表示させたいプロパティの内部値*
![](/images/bb90d880c8890c/2024-08-15-15-30-36.png)

:::details collection と そうでないものの違い
チュートリアルの説明を聞く限り collection はオブジェクト内の一覧。p_オブジェクト名 のほうはオブジェクトに対する個別レコードを指す時に使うっぽいです。
今回は一覧を出したいので collection を使います。
:::

クエリが表示されたら［コピー］をクリックしてどっかにメモっておくか、タブをそのまま寝かせておきます。

## 2.テーマを作成する
HubSpotのコンテンツ＞デザインマネージャーの画面にアクセスし、［ファイル］＞［新規テーマを作成］から新規テーマを作成します。
![](/images/bb90d880c8890c/2024-08-15-15-36-34.png)
特に今回は難しいことを考えずにボイラープレートを使いたいと思います。

https://developers.hubspot.jp/docs/cms/building-blocks/themes/hubspot-cms-boilerplate

## 3.クエリをテーマ内に設置
ステップ1で用意したクエリをステップ2で作ったテーマの中におきたいと思います。そのことで ステップ4のテンプレートから呼び出すことができるって感じですね

わかりやすいように専用のフォルダを作って
![](/images/bb90d880c8890c/2024-08-15-15-44-12.png)

新規ファイルを作成＞GraphQLのファイルを作成します
![](/images/bb90d880c8890c/2024-08-15-15-44-37.png)


ファイルができたらステップ1で作ったクエリをよいしょと貼り付けます。
![](/images/bb90d880c8890c/2024-08-15-15-46-53.png)

で、クエリのお名前もわかりやすいように "bands"とかにしてあげて、ラベルとディスクリプションも書いておきましょう。
![](/images/bb90d880c8890c/2024-08-15-15-47-46.png)


📖 ラベルとディスクリプションの書き方については[このドキュメント](https://developers.hubspot.com/docs/cms/data/use-graphql-data-in-your-website-pages#pass-dynamic-context-into-a-query)内のコードから拝借してます。

📍 bands-data.graphqlの中身
```html
# label: Our Bands info page
# description: "List of all bands"

query bands {
  CRM {
    p_bands_collection {
      items {
        _metadata {
          id
        }
        band_name
        best_sold_album
        description
      }
    }
  }
}
```

## 4.テンプレートを作成する
ではでは実際にバンドを表示するページのテンプレートを作成します。
ここではボイラーテンプレートにあるテンプレートを複製しても、新規ファイルからいじってもどちらでも問題ないです。

複製した場合はテンプレート内から `{% dnd_area %}` 一帯を消してください。
必要なパーツは以下の３つです：
```html
{% extends "./layouts/base.html" %}
{% block body %}
<!--ここにいろいろ書いていきます-->
{% endblock body %}
```
![](/images/bb90d880c8890c/2024-08-15-15-52-44.png)

## 5.テンプレートでクエリを呼び出す
テンプレートでクエリを呼び出していこうと思います (˘ω˘)

中身を書いていく前にlabelとかある部分にクエリの宣言を追加します。
`dataQueryPath: ../data-query/bands-data` みたいな感じで、GraphQLのファイルパスを書きます。（ここでは拡張子は不要です）
![](/images/bb90d880c8890c/2024-08-15-16-01-48.png)
*囲ったとこ*

で次に、"bands"はこのクエリを指しますのまじない🪄をかけます
```html
{% set bands = data_query.data.CRM.p_bands_collection.items %}
```
![](/images/bb90d880c8890c/2024-08-15-16-07-11.png)

後半部分が謎めいてますが、クエリにある { を拾っているイメージですね
![](/images/bb90d880c8890c/2024-08-15-15-59-37.png)

で、今回は一覧項目を繰り返し表示させたいので forループ🪄を書きます。
```html
{% for band in bands %}
{% endfor %}
```
![](/images/bb90d880c8890c/2024-08-15-16-09-09.png)

枠ができたら`for`の間に表示させたい項目（変数）を書いていきます。
例えばバンド名については`{{band.band_name}}`という形で、定義した名前とプロパティの内部値の順序で記述します。

📍 bands-listing.html（テンプレート）の中身
```html
<!--
    templateType: page
    label: About
    isAvailableForNewContent: true
    dataQueryPath: ../data-query/bands-data
-->
{% extends "./layouts/base.html" %}

{% block body %}

    <h1>
    Our Bands
    </h1>

    {% set bands = data_query.data.CRM.p_bands_collection.items %}
    {% for band in bands %}
    <h3>
    {{band.band_name}}
    </h3>

    <p>
    {{band.description}}
    </p>

    <p>
    {{band.best_sold_album}}
    </p>
    {% endfor %}
{% endblock body %}
```

すると右側の出力パネルにカスタムオブジェクトが呼び出されていることがわかります！
![](/images/bb90d880c8890c/2024-08-15-16-07-39.png)

:::details HubL参考記事
ちゃんとした書き方は以下ドキュメントをご参照ください📖
https://developers.hubspot.jp/beta-docs/reference/cms/hubl/variables-macros-syntax
https://developers.hubspot.jp/beta-docs/reference/cms/hubl/loops
:::

あとはページにテンプレートを適用させるなり、テンプレートをプレビューさせるなりしてちゃんと表示されているかをチェックしてかんりょーです！
![](/images/bb90d880c8890c/2024-08-15-16-12-59.png)

## おわりに
ほほーんって感じで作ってみましたが、クエリを書き換えるだけで表示できる項目を調整できるのは便利そうですね🤔

また 本来の目的としてはページ訪問者に対してパーソナライズされた内容を出すこと らしいので、
>GraphQL allows you to create data queries that access your HubSpot CRM and HubDB data to create personalized and data-driven experiences for your website visitors. [参照ドキュメント](https://developers.hubspot.com/docs/cms/data/use-graphql-data-in-your-website-pages)

非公開コンテンツとか訪問されているコンタクトの情報がわかっててそれ毎の内容を出し分けたいってときとかに使えるみたいです。
（そんな雰囲気なことが[ここ](https://developers.hubspot.com/docs/cms/data/use-graphql-data-in-your-website-pages#create-a-query-and-module-for-your-dynamic-pages)にかいてありました）
