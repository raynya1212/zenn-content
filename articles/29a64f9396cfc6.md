## 「新しい開発者プラットフォーム」…？
なにやら HubSpot の「非公開アプリ」がレガシー扱いになったという話を聞き、全然向き合えていなかったのですが …😑
ようやくドキュメントを読んでみたので、「**新しい開発者プラットフォーム**」なるものについてまとめて見たいと思います 📝

:::message
HubSpot Developers のドキュメントは日本語版の更新が遅いようなので、英語版の参照をすることをおすすめします 𓏼˙₊🕊️◝ྀ

### Developer platform とは
HubSpot の新しい開発者プラットフォーム (Developer platform) とは、アプリや CRM 開発のための統合された場所 ということのようです。

>HubSpot’s new developer platform, version 2025.2, provides multiple advantages over legacy private and public apps, including a new file-based build-and-deploy framework. This framework contains an app’s configuration, assets, and other source code. [Developer platform](https://developers.hubspot.com/docs/apps/developer-platform/build-apps/overview)

今までは 
- 社内で運用するための非公開アプリ
- アプリ マーケットプレイスに掲載する公開アプリ
- テーマ開発、サーバーレス関数のためのプロジェクト などなど…

色々とわかれていたので、それらをまとめて 「プロジェクト」にまとめて管理するよー🪄︎︎🫧
ということなのかな…？💭

### [開発] 画面を開いてみる
HubSpot の メニュー > [開発] から [開発者概要] (Developer overview page) 画面を開くことができます。

📄∘˙ [開発] 画面にアクセスするには、以下の権限 / シートが必要です （開発者シート なんてのもできたんですね…💭）
> Super admins, partner admins, and users with either a developer seat or the Developer tools access permission can access the developer overview page. [Developer overview](https://developers.hubspot.com/docs/getting-started/developer-overview)

Developer overview page を開くと、 [**プロジェクト**] というメニューがあります。

## App cards をつくってみよう
それでは、Developer platfrom を使って何かつくってみることにします 🪄༘✩

### App cards について
今回は、コンタクトや会社レコード画面に表示できる「カード」のカスタム開発にトライしてみます
![](/images/29a64f9396cfc6/2025-12-29-20-50-40.png)

このレコード画面のカード開発、過去には [UI extensions](https://github.com/HubSpot/ui-extensions-examples) と言われていた気がするのですが
現在は **"App cards"** とよばれているようです

https://developers.hubspot.com/docs/apps/developer-platform/add-features/ui-extensibility/app-cards/overview

App cards については、👇のチュートリアルがあったので、これをもとにためしてみることにします 𓈒⁺🕊️⸝⸝
https://developers.hubspot.com/docs/getting-started/quickstart

### 1. HubSpot CLI をセットアップする
HubSpot のローカル開発環境である [HubSpot CLI](https://developers.hubspot.com/docs/developer-tooling/local-development/hubspot-cli/install-the-cli) を作業ディレクトリにインストールします

VS code での使い方については👇にまとめているので、VS code 使いな方はこちらの方法で進めるのがおすすめです ꕀ🐶ˊ˗
https://zenn.dev/raynya/articles/hubspot-vscode-cli

:::message
今回のチュートリアルを進める上では、`7.6.0` バージョン以上がインストールされている必要があります。

ただ、👇の記載にもあるように、できることなら最新の `7.11.2` をインストールしておくのがいいようです：
> The latest version of the HubSpot CLI, which is recommended, is 7.11.2. You can check which version of the CLI you have installed by running hs --version. [Install the HubSpot CLI](https://developers.hubspot.com/docs/developer-tooling/local-development/hubspot-cli/install-the-cli)


既に HubSpot CLI が導入されている環境だったら、
```bash
hs --version
```
で、バージョンが `7.6.0` 以上かを確認してみてください

最新のバージョンに更新するには
```bash
npm i -g @hubspot/cli@latest
```
コマンドを使います 🐈‍⬛⸝⋆
:::

### 2. プロジェクトを作成・アップロードする
1. HubSpot CLI のインストールが完了したら **`hs get-started` コマンド**を使って、サンプルプロジェクトの作成を進めます

```bash
hs get-started
```

:::message
プロジェクトは、[`hs project create` で作成](https://developers.hubspot.com/docs/developer-tooling/local-development/hubspot-cli/project-commands#create-a-new-project)することも可能です

 `hs get-started` を使うと
>We'll create a new project with a sample app for you.Projects are what you can use to create apps with HubSpot.Usually you'll use the `hs project create` command, but we'll go ahead and make one now.

と言ってくるので、

- 用途が決まっている場合は `hs project create`
- とりあえずデプロイじゃ！なときは `hs get-started` 

って感じで使い分けるのがよさそう 🐰 ∘° 
:::

1. `Are you looking to build apps or CMS assets?` と聞かれるので `App` を選択
2. `Enter your project name` と聞かれるので、プロジェクト名を入力します（任意の値。無難に `app_cards-quicksart` にしてみます）
3. `Choose where to create the project` と聞かれるので、プロジェクトを置きたいディレクトリを指定します（作業ディレクトリを開いている場合は、そのまま Enter で OK）

![](/images/29a64f9396cfc6/2025-12-29-21-28-08.png)

4. `? Would you like to upload this project to account <ポータル名>` と聞かれるので、`Y` を押下してアップロードしちゃいましょう
5. HubSpot CLI が ビルド＆デプロイを頑張ってくれる 🚀 ⌢ ۪۫ので、見守ります

`[SUCCESS] Project uploaded successfully!` と表示されたらアップロードまで成功です！˗ˏˋ₊ʚ🌟ɞ₊ˎˊ˗
![](/images/29a64f9396cfc6/2025-12-29-21-35-01.png)

### 3. アプリをインストールして、確認！

1. HubSpot の [開発] > [プロジェクト] で今回作成したプロジェクトがあることを確認します

![](/images/29a64f9396cfc6/2025-12-29-21-38-30.png)

2. 作成したプロジェクトを開くと「My Get Started app」というアプリが作成されていることがわかります

📄∘˙ これが今まで非公開アプリにあたるものだったと思うのですが、Developer platform では「プロジェクトコンポーネント」というようですね
![](/images/29a64f9396cfc6/2025-12-29-21-41-44.png)

3. プロジェクト > アプリ を開き、[配信] (Distribution) タブ > [今すぐインストール] からアプリのインストールを行います
![](/images/29a64f9396cfc6/2025-12-29-21-40-32.png)

📄∘˙ こんな画面が出てきますが、自分が作っているアプリなので [アプリを接続] で進めちゃいます
![](/images/29a64f9396cfc6/2025-12-29-21-47-50.png)

[接続されたアプリ] の一覧で「My Get Started app」のアプリが表示されていればインストール完了です！˗ˏˋ₊ʚ🌟ɞ₊ˎˊ˗
![](/images/29a64f9396cfc6/2025-12-29-21-49-20.png)

### 4. コンタクトレコードにカードを追加してみよう
1. アプリのインストールが完了したら、`hs project dev` コマンドを入力し、ローカル開発サーバーを立ち上げます
```bash
hs project dev
```
![](/images/29a64f9396cfc6/2025-12-29-21-59-32.png)

:::message
このときコマンドはプロジェクトのディレクトリに移動したうえで実行しましょう 🐰 ∘° 
あと、推奨じゃないみたいだけど、今回は本番環境にそのまま立ち上げています
:::

2. ローカル開発サーバーが立ち上がると、ブラウザで👇こんな画面がでてきて、`Connected` な状態であることが表示されます

![](/images/29a64f9396cfc6/2025-12-29-22-01-34.png)

3. [Setup preview] をクリックすると、[接続されたアプリ] >「My Get Started app」アプリの設定画面？[アプリカード] タブに遷移します。ここで、カードをレコードページに表示させるように設定しておきましょう
![](/images/29a64f9396cfc6/2025-12-29-22-04-23.png)

:::message
私の環境にはビューがデフォルトのしかないので、👆の見た目になっていますが [設定] > [コンタクト]> [レコードのカスタマイズ] でビューを作っている場合は、任意のビューが選べるはずです 🐰 ∘° 
:::

4. ビューの設定画面を開くと、追加した「Get Started App Card」が追加されていることがわかります
※ みやすいように位置調整しました。3 の手順で追加すると、最下部にカードは追加されます
![](/images/29a64f9396cfc6/2025-12-29-22-08-49.png)

5. またコンタクトレコードを開くと、追加したカードが表示されていますね ⊹˙∘•💫ᯓ （わーい！）

![](/images/29a64f9396cfc6/2025-12-29-22-11-20.png)

📄∘˙ `ローカルでの開発` の黄色いタグは、ローカル開発サーバーが立ち上がっている状態だよ～とお知らせしてくれるもののようです

> You’ll then be redirected back to the contact record, and the app card will display with a Developing locally tag to indicate that the local development server is waiting to detect local changes. [Setup local development preview](https://developers.hubspot.com/docs/getting-started/quickstart#install-and-preview-your-app)

### 5. カードをすこしカスタムしてみよう
せっかくなので、作成したカードをすこしカスタマイズしてみましょう ‧₊ 🍥 ˚✩

App cards のプロジェクトは以下の構成で作られています：
```
myProject
└── src/
    └── app/
        └── cards/
            ├── NewCard-hsmeta.json
            ├── NewCard.jsx
            └── package.json
```
このうち、`jsx` のファイルがカードの見た目 🪞⟡ を構成している部分なので、ここに当たるファイルを少しいじってみましょう

1. プロジェクトを作成したディレクトリから `app_cards-quickstart\src\app\cards\get-started-app-card.tsx` をみつけます
2. `get-started-app-card.tsx` は以下の構成になっていたので、今回はテキストを追加してみることにします

```diff tsx
import React from "react";
import { Text, Link, List, EmptyState } from "@hubspot/ui-extensions";
import { hubspot } from "@hubspot/ui-extensions";

hubspot.extend<'crm.record.tab'>(() => <Extension />);

const Extension = () => {

  const appCardDocsLink = 'https://developers.hubspot.com/docs/apps/developer-platform/add-features/ui-extensibility/app-cards/overview';

// return 以下がカードに表示される内容
  return (
    <>
      <EmptyState
        title="Build your app card here!"
        layout="vertical"
        imageName='building'
      >
+       <Text>
+       App Card を作ってみたよ🐈！
+       </Text>
        <Text>
          Add a layer of UI customization to your app by including app cards that can display data, allow users to perform actions, and more.
          Visit the <Link href={appCardDocsLink}>app card documentation</Link> for more info, or check out the following links to get inspired:
        </Text>
        <List variant="unordered-styled">
          <Link href="https://developers.hubspot.com/docs/platform/ui-components">
            📖 Explore our library of UI components
          </Link>
          <Link href="https://ecosystem.hubspot.com/marketplace/apps/app-cards">
            📖 Look at the Marketplace collection of apps that contain app cards
          </Link>
          <Link href="https://developers.hubspot.com/slack">
            ▶️ Connect with developers on #ui-extensions channel on developer
            Slack community
          </Link>
        </List>
      </EmptyState>
    </>
  );
};
```

3. 編集しファイルを保存 > コンタクトレコード画面をリロードすると、カードに追加したテキストが表示されていることがわかります ⊹˙∘•💫ᯓ （わーいわーい！）

![](/images/29a64f9396cfc6/2025-12-29-22-31-03.png)

📄∘˙ ローカル開発サーバーが立ち上がっているから、**自動的に変更がプレビューとして反映**されるということですね

:::message alert
👆の状態は、ローカルで編集した内容が、プレビューされている状態になります。
なので、**変更を本番環境に反映させるには次のステップにある `hs project upload` を忘れず行いましょう** 𓂃  ໋ 🚀
:::

4. 変更した追加点を本番環境に反映させるべく、`hs project upload` を実行します

```bash
hs project upload
```
![](/images/29a64f9396cfc6/2025-12-29-22-40-27.png)

:::message
このときローカル開発サーバーは `q` で終了させておきましょう 🐰 ∘° 
:::

5. ビルド -> デプロイ -> アップロードが完了したら、先ほど確認していたコンタクトレコード画面をリロードして変更が反映されているかを確認します

📄∘˙ さっきまであった `ローカルでの開発` のタグがなくなっていて、追加したテキストが表示されていることがわかります
👉 無事本番環境に反映させることができました ⊹˙∘•💫ᯓ （わーいわーいわーい！）

![](/images/29a64f9396cfc6/2025-12-29-22-41-27.png)
